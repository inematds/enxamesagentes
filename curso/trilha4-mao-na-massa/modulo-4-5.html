📋 Modo Prometheus: entrevista antes de executar

A entrevista previa faz toda a diferenca — Prometheus extrai requisitos completos antes de uma linha de codigo.

📋 Por que a entrevista faz toda a diferenca

No modulo anterior, voce fez um pedido direto e o Sisyphus executou. Funciona para tarefas simples, mas para projetos maiores (uma landing page completa, um app, um dashboard), a qualidade cai drasticamente se voce nao definir bem o escopo. O Prometheus resolve isso: ele faz uma entrevista estruturada que extrai todos os requisitos antes de escrever uma unica linha de codigo.

💡 Discovery: a fase mais importante

  • •Sem entrevista: O swarm "adivinha" o que voce quer — resultado generico, muitos ajustes depois
  • •Com entrevista: O swarm sabe exatamente o que voce quer — resultado personalizado, poucos ajustes
  • •Na pratica: 5 minutos respondendo perguntas economiza 30+ minutos de ajustes depois
2

🔄 Passo 1: trocar para o agente Prometheus

O Prometheus nao e ativado automaticamente — voce precisa mudar para ele manualmente. Isso e intencional: o modo padrao (Sisyphus) e otimizado para tarefas diretas. O Prometheus e para quando voce quer qualidade maxima.

💡 Como ativar o Prometheus

# Dentro do OpenCode, pressione Tab
# Uma lista de agentes aparece:
#   Sisyphus (padrao)
#   Hephaestus
#   Prometheus / Plan Builder  ← selecione este
#   Plan Executor

# Selecione "Prometheus" ou "Plan Builder"
# O indicador no topo do chat muda para Prometheus
# Agora voce esta conversando com o planejador!
3

✍️ Passo 2: descrever o projeto e pedir entrevista

Com o Prometheus ativo, descreva o que voce quer construir e explicitamente peca que ele te entreviste primeiro. O Prometheus vai fazer perguntas estruturadas para entender completamente o escopo.

💡 Prompt para iniciar a entrevista

# Exemplo de prompt para o Prometheus:

Quero construir uma landing page profissional para uma
consultoria de IA. Me entreviste antes de comecar.

# O Prometheus vai responder com perguntas como:
# - Qual o nome da empresa?
# - Quais servicos voce oferece?
# - Quem e o publico-alvo?
# - Que secoes a pagina deve ter?
# - Qual o tom de voz (formal, informal, tecnico)?
# - Qual stack tecnico preferido?
# - Precisa de formulario de contato?
# - Tem referencias visuais?
4

🗣️ Passo 3: responder as perguntas do Prometheus

Cada resposta que voce da refina o plano. Quanto mais contexto e detalhes voce fornece, melhor sera o resultado final. Nao economize nas respostas — este e o momento em que voce molda o projeto.

💡 Exemplo de respostas

# Prometheus pergunta: "Qual o nome da empresa e servicos?"
# Sua resposta:
A empresa se chama "SoluTech IA". Oferecemos consultoria em
automacao com IA, desenvolvimento de chatbots, e treinamentos
corporativos sobre agentes de IA.

# Prometheus pergunta: "Quais secoes quer na pagina?"
# Sua resposta:
Hero com titulo impactante, secao de 6 servicos com icones,
como trabalhamos (4 etapas), depoimentos de 3 clientes,
FAQ com 5 perguntas, formulario de contato e footer.

# Prometheus pergunta: "Stack e tom de voz?"
# Sua resposta:
HTML, CSS e JavaScript vanilla. Tema escuro. Tom profissional
mas acessivel, sem ser informal demais.

📊 Categorias tipicas de perguntas

  • Identidade: Nome, logo, cores, marca
  • Conteudo: Servicos, secoes, textos, depoimentos
  • Tecnico: Stack, frameworks, responsividade
  • Tom: Formal, casual, tecnico, inspiracional
  • Funcionalidade: Formularios, animacoes, links
5

📊 Passo 4: revisar o resumo e confirmar

Apos todas as perguntas, o Prometheus compila tudo em um resumo estruturado. Revise com cuidado — esta e sua ultima chance de corrigir antes da execucao.

💡 O resumo do Prometheus

# Prometheus mostra algo como:
#
# RESUMO DO PROJETO
# ─────────────────
# Nome: SoluTech IA
# Tipo: Landing page profissional
# Servicos: Consultoria IA, Chatbots, Treinamentos
# Secoes: Hero, Servicos(6), Processo(4), Depoimentos(3),
#          FAQ(5), Formulario, Footer
# Stack: HTML + CSS + JS vanilla
# Tema: Escuro
# Tom: Profissional acessivel
# Formulario: nome, email, empresa, telefone, mensagem
#
# "Confirma? Arrancamos com o plano de implementacao?"

# Sua resposta:
Sim, correto. Pode comecar!
6

🚀 Passo 5: executar com /start-work

Apos confirmar, o plano esta pronto. Agora voce aciona a execucao com o comando /start-work. O Sisyphus assume o controle e comeca a delegar tarefas para todos os agentes em paralelo.

💡 Lancando a execucao

# Digite no chat:
/start-work

# O que acontece:
# 1. Sisyphus assume o controle
# 2. Plan Executor ativa as waves de implementacao
# 3. Wave 1: Estrutura HTML base
# 4. Wave 2: Conteudo de cada secao
# 5. Wave 3: Estilizacao CSS
# 6. Wave 4: JavaScript e interatividade
# 7. Wave 5: Revisao automatica
# 8. Wave 6: Polish final
#
# Agentes trabalham em PARALELO!
# Voce nao precisa fazer mais nada ate terminar.

📊 O poder do planejamento previo

Compare os resultados: a landing page criada com entrevista Prometheus tera secoes personalizadas, conteudo especifico da sua marca, tom de voz correto e todas as funcionalidades que voce pediu. Sem entrevista, o resultado seria generico com placeholder "Lorem ipsum". A diferenca e dramatica.
